cocos Creator学习-阅读文档与总结

Posted on 2016-04-09 | Edited on 2016-04-13 | In cocos Creator

环境

在MAC环境下, 使用Cocos Creator进行开发

注意事项

  1. 在编码的时候直接查看Cocos2d-JS的API
  2. 希望将旧的Cocos2d-JS代码直接在Creator上进行运行

已上是需要注意的地方, 根据官方的解释, 是因为

Cocos Creator 可以说脱胎自 Cocos2d-JS，它们的 API 一脉相承，有很多相同的部分，但由于使用了全新的组件化框架，两者的 API 是有差异的，并且无法互相兼容

节点和组件

Cocos Creator 的工作流程是以组件式开发为核心的，组件式架构也称作 组件-实体系统（或 Entity-Component System），简单的说，就是以组合而非继承的方式进行实体的构建。

理解

各个控件, 单元之间, 并非以继承的关系进行关联, 而是组合, 就如同有一些零件, 实例根据需求, 按需索取, 组装成想要的样子. 从而很好的解决了多继承以及类爆炸的问题

坐标系(Coordinate)

分类

  1. 笛卡尔坐标系 - 右手
  2. 屏幕坐标系
  3. 世界坐标系 - 绝对
  4. 本地坐标系 - 相对父类

锚点(Anchor)

锚点（Anchor） 是节点的另一个重要属性，它决定了节点以自身约束框中的哪一个点作为整个节点的位置。我们选中节点后看到变换工具出现的位置就是节点的锚点位置。

理解

位于场景根级别的节点, 参考系为场景. 非位于场景根级别的节点的位置则根据父节点的锚点加上本身的position进行偏移

启示

  1. 在节点的布局上来说, 尽量让有关系的节点放到同一个父级上, 而不是直接放到场景中. 这样他们的坐标参考系有关联, 便于操作
  2. 在渲染的时候, 子节点会覆盖住父节点

生命周期

  1. 触发时间
  2. 应用场景
  3. more

onLoad

  • 组件所在场景被加载时触发
  • 用于初始化
  • 可获取场景内其他节点

start

  • 第一次执行update之前触发
  • 初始化一些中间状态的数据

update

  • 渲染前
  • 更新物体行为与属性

lateUpdate

  • 所有组件update完成后执行
  • 希望在update之后执行一些其他操作

onEnable

  • enable 从false 变为 true

onDisable

  • enable true 变为 false

onDestory

  • 销毁收回
