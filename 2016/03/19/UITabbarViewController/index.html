UITabbarViewController

Posted on 2016-03-19 | Edited on 2016-03-25 | In iOS

前言

为了更加熟悉苹果API, 所以抽空看一下官方文档, 并随之做一下记录, 仅仅是个人见解, 如有遗漏或不对的地方, 请留言

UITabbarViewController

这个控件在开发过程中用的非常多, 所以先从这个入手. 值得一提的是, 苹果的文档介绍的顺序根据使用频度和重要性进行排序的.

delegate

其实这个看上去就知道, 是一个代理, 并没有什么好说的, 真正需要看的, 是代理所遵循的协议, UITabBarControllerDelegate, 稍后我们在看这个协议

tabbar

var tabBar: UITabBar { get }

这是一个只读属性的, 所以在项目中, 如果我们想做点什么, 那就只好使用KVC来进行替换了.

viewControllers

var viewControllers: [UIViewController]?

The default value of this property is nil. When configuring a tab bar controller, you can use this property to specify the content for each tab of the tab bar interface. The order of the view controllers in the array corresponds to the display order in the tab bar. Thus, the controller at index 0 corresponds to the left-most tab, the controller at index 1 the next tab to the right, and so on. If there are more view controllers than can fit in the tab bar, view controllers at the end of the array are managed by the More navigation controller, which is itself not included in this array.

这是官方的描述, 说这个属性默认是空, 你可以使用这个属性去获得tabbarController上的每一个控制器, 顺序你和添加的顺序一样, 如果你添加了很多的vc在上面, 这个array会托管给more vc, 在这里我说一句, 一般超过五个, 末尾就会变成more, 这可不是一个好的交互

func setViewControllers(_ viewControllers: [UIViewController]?, animated animated: Bool)

When you assign a new set of view controllers at runtime, the tab bar controller removes all of the old view controllers before installing the new ones. When changing the view controllers, the tab bar controller remembers the view controller object that was previously selected and attempts to reselect it. If the selected view controller is no longer present, it attempts to select the view controller at the same index in the array as the previous selection. If that index is invalid, it selects the view controller at index 0.

当你在运行的时候给view controllers赋新值的时候, tabbarController 会把旧的全都移除, 然后放置心的. 当改变vcs的时候, tabbarController会记住当前所选择的vc, 如果那个vc不存在了, tabbarController会尝试找同索引的vc, 如果还是没有, 那他就会去选择第一个了.

More

var customizableViewControllers: [UIViewController]?
var moreNavigationController: UINavigationController { get }

这两个属性, 都是在控制器多于5个后, 出现了MORE选项而产生的, 鉴于这种情况几乎用不到, 就此略过

Selected

unowned(unsafe) var selectedViewController: UIViewController?
var selectedIndex: Int

这两个分别是选中的VC和索引, 值得注意的是, selectedViewController的修饰符不是weak, 个人认为这是历史遗留问题

UITabBarControllerDelegate

下面让我们看看协议

optional func tabBarController(_ tabBarController: UITabBarController, shouldSelectViewController viewController: UIViewController) -> Bool

很显然, 这个代理是控制某一个控制器是否能被激活

optional func tabBarController(_ tabBarController: UITabBarController, didSelectViewController viewController: UIViewController)

这个是选中某一个controller以后触发的方法

More

optional func tabBarController(_ tabBarController: UITabBarController, willBeginCustomizingViewControllers viewControllers: [UIViewController])

optional func tabBarController(_ tabBarController: UITabBarController,willEndCustomizingViewControllers viewControllers: [UIViewController], changed changed: Bool)

optional func tabBarController(_ tabBarController: UITabBarController, didEndCustomizingViewControllers viewControllers: [UIViewController], changed changed: Bool)

这三个代理方法都是在出现More的情况下, 点击More, 你可以看到右上角有个Edit按钮, 编辑的时候会触发的

UIInterfaceOrientationMask vs UIInterfaceOrientation

optional func tabBarControllerSupportedInterfaceOrientations(_ tabBarController: UITabBarController) -> UIInterfaceOrientationMask

optional func tabBarControllerPreferredInterfaceOrientationForPresentation(_ tabBarController: UITabBarController) -> UIInterfaceOrientation

这两个和方向有关的代理, UIInterfaceOrientationMask是UIInterfaceOrientation的一个集合, 但是我并没有发现怎么才能调用第一个方法.

动画

optional func tabBarController(_ tabBarController: UITabBarController, animationControllerForTransitionFromViewController fromVC: UIViewController, toViewController toVC: UIViewController) -> UIViewControllerAnimatedTransitioning?

optional func tabBarController(_ tabBarController: UITabBarController, interactionControllerForAnimationController animationController: UIViewControllerAnimatedTransitioning) -> UIViewControllerInteractiveTransitioning?

很遗憾的告诉你, 我对这两个方法并不熟悉, 我猜想是控制器的转场动画, 当然, 我会在后续的博客解读他
