通过可点击文字看CoreText

Posted on 2016-08-27 | Edited on 2016-09-23 | In iOS

起因

前一段时间自学了一丢丢Android, 那时候看到Android的Demo中实现可点击文字, 惯性的想了想iOS怎么实现, 于是了解了一下CoreText, 本来想愉快的学完把两个平台的都总结一下, 结果工作如泰山压顶般的压了下来, 过了好一阵子才开始总结. 已无力吐槽了, 撸码好了. 这里有一个比砂纸还粗糙的Demo, 需要的朋友可以down一下.

Framework

查了很多资料, 都没说明白iOS怎么实现可点击文字, 但都说需要用CoreText. 这时, 突然想起以前看过的一个很NB的框架YYKit, then I run the demo, 果然可以, 所以想潜心看看源码是如何做到的. 对于平时我们用到的一些控件, 先看看下图, 以便理解NSAttributedString.

为什么说这个图和NSAttributedString有关系呢, 继续往下看.

NSAttributedString

实现思路

这里需要提到我们的基础–属性文字, 在此之前, 我对它的理解就是可是设置各种样式的一种文字, 看了YYKit之后, 我觉得我理解的太肤浅了, NSAttributedString应该是一个带有KV对的文字, 而我们之所以能用KV对去给文字设置属性, 是因为系统底层, 对对应的Value取出并加以加工, 我们可以自定义一个KEY, 它的VALUE是一个function pointer, 然后在需要的时候取出去调用, 这就是可点击文字的实现的大概思路.

和上面图的关系

那和上面的那个图有什么关系呢? 我觉得iOS整体就是一层一层的对底层逐步的封装, 而在平时我们写代码的时候, 也会有一些封装, 所以这些控件什么的, 它并不是不可拆解的, 它是有底层功能组装起来的. 我们接下来的第一个例子是图文混排. 让我们先热热身, 看看NSAttributedString是怎样的一个KV对的文字. 不过在这之前, 我们需要准备点基础知识

Core Text

CT

让我们先来看一张图

简单的来说系统将AttributeString -> frameSetter -> CFFrame, CFFrame是由许多的CTLine组成, 而CTLine又由许多的CTRun组成. in an other word, CTRun是属性文字中最小的单位. 我们接下来要这样做, 在一段文字中, 插入一个空字符, 作为某一个图片的占位符, 然后为这个字符设置一些KV对, 在另一端, 我们取出这些KV对, 做对应的事情.

Let’s code

首先我们写一个CTRunDelegateCallbacks, 这个作用是, 系统在将占位符换成图片的时候, 需要知道怎么把这个图片画到上面, 具体的, 有上行间距, 下行间距, 宽度.

接下来, 我们要在当前的内容中插入一个Attribute空字符, 然后为这个空字符绑定两个属性, 一个是CTRunDelegate, 以保证程序会调用我们刚刚创建的代理. 然后我们自定义了一个AttributeName, 为KEY, VALUE是图片的名字, 当然这里我偷懒

接下来要说明的是, UIKit和CoreText的坐标系的问题

至于为什么是这样, 我猜想CoreText是从OS系统搬过来的. 那么我们在操作的时候, 就要做点什么让所有操作处于同一个坐标系下.

当然, 这里需要先自定义一个Object来存放function pointer, 因为在给attributeString设置属性的时候, value是AnyObject, 很显然, function pointer是不合适的. 那么, 我们就封装一下.
