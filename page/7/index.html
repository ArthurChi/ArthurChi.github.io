iOS开发过程中遇到的cookie的一些操作

Posted on 2016-09-05 | In iOS

起因

在WebView页面种下cookie后, 不知什么原因, 再请求网页的时候, 并没有带上, 查了好多资料, 都说iOS在种和获取cookie方面比Android要简单的多, 会自动带上, 但是没有找到问题出在哪里, 于是就想出了一个曲线的解决方法.

需求

要求在登录的时候, 调用某一个网页, 该网页会set-cookie, 然后在点击其他页面的时候, 会自动带上, 在登出后要将cookie清除. 由于其他功能对于cookie没有依赖, 所以做了个简单粗暴的解决方式

Code

这里我使用的是WKWebView, 具体方式就是在登录的时候, 将response的头中的cookie取出, 然后设置给全局的cookie存储对象, 然后在登出的时候, 进行删除. 当然, 如果需要精准控制的, 可以使用URL作为KEY去存储和删除.
