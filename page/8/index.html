点击按钮叨叨叨(二)

Posted on 2016-04-06 | Edited on 2016-04-10 | In iOS

前言

上一篇, 我们说了响应者链条, 这篇我们说说按钮点击事件, 咱们还是按照惯例, 看看文档

UIControl

Buttons use the target-action design pattern to notify your app when the user taps the button. Rather than handle touch events directly, you assign action methods to the button and designate which events trigger calls to your methods. At runtime, the button handles all incoming touch events and calls your methods in response.

当用户点击了按钮的时候, 按钮采用了target-action的设计模式去通知你的app. 比起响应touch events更加直接, 你指定一个方法, 然后指定哪种事件会触发你的方法. 在运行时, 按钮会捕捉你所有的输入的touch事件, 然后调用你的方法作为响应.

他的意思是这个方式所添加的事件响应的优先级高于UIResponse的那些touch触发事件. 接下来让我们看看按钮的点击过程.

事件处理

根据上一篇所说的响应者链条, 找到最合适的View, 假设这里找到的是一个按钮, 我们之前通过addtarget的方式给它添加了一个响应事件, 那他是如何找到这个方法的呢? runloop会在app启动的时候, 将所有的方法哈希化以后, 组成selector的一个set, 这也就是为什么OC不允许方法重载的原因, 因为你的方法签名经过哈希后会有冲突. 系统根据你发送的这个消息, 到这个set里面查找, 发现有, 于是就他会根据分配的target对象, 到他的methodList里面去找对应的IMP, 找到了, 执行. 如果找不到, 则进行消息转发. 这样, 事件处理完成了.

结束

至此, 一个简单的按钮点击处理过程就结束了. 其实分裂开来看是很简单的, 重要的是把他们和运行时结合起来, 系统的看待这个过程.

  2. 未插入状态

前言

为了更加熟悉苹果API, 所以抽空看一下官方文档, 并随之做一下记录, 仅仅是个人见解, 如有遗漏或不对的地方, 请留言

UITabbarViewController

这个控件在开发过程中用的非常多, 所以先从这个入手. 值得一提的是, 苹果的文档介绍的顺序根据使用频度和重要性进行排序的.

delegate

其实这个看上去就知道, 是一个代理, 并没有什么好说的, 真正需要看的, 是代理所遵循的协议, UITabBarControllerDelegate, 稍后我们在看这个协议

tabbar

var tabBar: UITabBar { get }

这是一个只读属性的, 所以在项目中, 如果我们想做点什么, 那就只好使用KVC来进行替换了.

viewControllers

var viewControllers: [UIViewController]?

The default value of this property is nil. When configuring a tab bar controller, you can use this property to specify the content for each tab of the tab bar interface. The order of the view controllers in the array corresponds to the display order in the tab bar. Thus, the controller at index 0 corresponds to the left-most tab, the controller at index 1 the next tab to the right, and so on. If there are more view controllers than can fit in the tab bar, view controllers at the end of the array are managed by the More navigation controller, which is itself not included in this array.

这是官方的描述, 说这个属性默认是空, 你可以使用这个属性去获得tabbarController上的每一个控制器, 顺序你和添加的顺序一样, 如果你添加了很多的vc在上面, 这个array会托管给more vc, 在这里我说一句, 一般超过五个, 末尾就会变成more, 这可不是一个好的交互
