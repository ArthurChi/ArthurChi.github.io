Runtime

这是一个宽广的话题, 这里我只是给自己放一个记忆的提示板, 以便以后复习.

定义

动态特性, 在C语言基础上, 维护了一套能够动态的运行系统, 该系统能够对该系统封装的类进行一些操作

layout

在谈到运行时的时候, 我们粗略的将对象的内存布局分为以下的几个点:

  1. ISA指针
  2. mate class
  3. super class
  4. 方法缓存列表
  5. 属性列表
  6. 方法列表
  7. 协议列表

具体关系如下图

Relationship

在这幅图中, 需要注意的是:

  1. 每个类的ISA都指向meta, meta class 的isa指针指向root meta class
  2. NSObject的super class是nil
  3. RootClass 的 super class 是Root Class
  4. RootClass的元类的ISA指向自己

对象的实例方法调用时，通过对象的 isa 在类中获取方法的实现。
类对象的类方法调用时，通过类的 isa 在元类中获取方法的实现。

题目

这篇文章, 权且把它当做是这篇文章的一个读书笔记了. 这篇文章, 说实话, 我也并没有完全吃透, 只能说有个大概的印象, 至于最后的三道题, 好好的解出来, 也着实花了一些功夫.

题目中自己要注意的

第一题, Father与Son其实都没有实现class方法, 那个get_class方法, 是NSObject的实现而已.
第二题, for循环中, 很巧妙的设置的初值, 值得借鉴
第三题, 需要注意的是, 栈地址是由高到低的进行分配的, 而C语言的struct是由低向高进行分配的, 所以造成了图中所示的那种压栈情况.
