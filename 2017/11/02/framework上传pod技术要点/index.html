framework上传pod技术要点

Posted on 2017-11-02 | Edited on 2017-11-30 | In iOS

Begin

我的环境是MAC

podSpec file

对于podSpec文件中, 需要注意的是 s.source_files 以及 s.resource_bundles都注释掉, 添加 s.vendored_frameworks 项. s.vendored_frameworks项的value往后看

http server

把自带的apache启动起来, 找到对应的Document目录

打包

我使用的是pod lib的模板进行创建的项目, 项目写好以后,

  1. 选择模拟器进行build
  2. 将得到的对应的framework拷贝到一个指定的目录下面
  3. 选择Generic iOS Device进行build
  4. 将得到的对应的framework拷贝到一个指定的目录下面, 当然, 要起个别的名字, 否则和第一次拷贝进来的文件会重名
  5. 参考这篇文章的最后那块脚本, 只需要lipo -create那块就可以了
  6. 然后将得到的文件拷贝回任一framwork下, 进行替换.
  7. 保险起见, 验证一下, 还是这篇文章中的 lipo -info命令
  8. 验证通过, 在项目的podSpec的目录下, 新建一个目录, 例如我这里叫frameworks, 将刚加工好的framework拷贝进来. 然后将frameworks这个目录进行压缩
  9. 将得到的压缩包放到apache的Document目录下面
  10. 使用浏览器尝试下载, 然后将路径拷贝出来.
  11. 如下图所示修改podSpect文件, 注意source和s.vendored_frameworks
  12. 最后, 重要的一点, zip文件中的文件, 包括目录, 要复制一份放到podsepc文件的同级目录下, in this case is framworks/SinaBookSDK.framework

然后就可以上传了, 至于是不是私库, 你决定, 开心就好.

后期补充

这里的http服务器是本地的, 所以, 如果你要提供给别人用, 就要放到一个公网环境上!
