QoS of GCD

有五种Qos, 他们分别是:

  1. User Interactive
  2. User Initiated
  3. Default
  4. Utility
  5. Background

User Interactive

Is this work actively involved in updating the UI?

需要立刻更新UI的, 换句话说, 就是需要在主线程中去进行执行的
例如, 画笔的绘制过程.

User Initiated

Is this work required to continue user interaction?

由用户触发, 但是不是要在主线程中进行执行的.
例如, 用户点击cell后, 需要准备显示的数据, 准备的过程, 可以使用这个优先级

Utility

Is the user aware of the progress of this work?

一些耗时操作, 但是同时, 用户对于这个事件的过程和进度是关心的.
例如, 网络 | I/O

Background

Is the user unaware of the work?
静默操作
例如, 垃圾处理

Flags

  1. DISPATCH_BLOCK_INHERIT_QOS_CLASS
  2. DISPATCH_BLOCK_ENFORCE
  3. DISPATCH_BLOCK_DETACHED

对于队列的优先级, 我就不用描述了, 这里着重说一下任务(block)的优先级和队列优先级的配合

当使用 DISPATCH_BLOCK_INHERIT_QOS_CLASS 修饰block的时候, 如果block的优先级高于queue的, block将被赋予队列相同的优先级.

当使用 DISPATCH_BLOCK_ENFORCE 修饰block的时候, 如果block的优先级高于其他已存在的block的优先级, 那么就会产生优先级反转. 为了解决这样的问题, 队列的优先级将被提高至block同样的高度或者将block优先级降至队列同等的高度.

当使用 DISPATCH_BLOCK_DETACHED 修饰block的时候, 则说明, 当前任务对于QoS并不关心.

Framework

https://github.com/ibireme/YYDispatchQueuePool

为什么而生

首先要明确 队列和线程的关系, 一个并行队列可以开的线程数量上限, 远远超过内存和CPU的承受力. 因此, 在QueuePool中, 使用结构体(context), 持有队列数组, 为每一种QoS的Queue开辟一块内存. 存放一种串行队列. 串行队列的个数和CPU的核数一样.

What’s we learned?

  1. 在程序中, 使用QoS来标记操作的优先级.
  2. 注意并行队列的并发
  3. DISPATCH_BLOCK_ENFORCE 注意优先级反转
