Swift中TableViewCell泛型的小坑记录

Posted on 2017-01-17 | In iOS

问题

由于自己搭建了一个急于公司业务的 快速开发的 小框架, 里面用到了泛型, 具体是这样的

  • 为cell建立了一个泛型, 用于描述cell所展示的模型.
  • 监听模型的didSet
  • didSet后, 给UI填充数据
  • 外界用过systemLayoutSizeFitting方法去获取高度, 并缓存

我这个出问题的cell使用xib建立, 然后关联类, 但是注册后运行, 总是提示Unknown class in Interface Builder file, WTF? 这是怎么回事.

转机

由于之前装x, 为这个项目封装过一个singleCellViewController, 目的是快速集成单一cell的listView, 让工程师刚专注于cell和model的对应关系, 而不是那些公共的, 配置参数请求和写controller上面. 这个vc里面就可以愉快的玩耍, 而这两个的区别不在于cell, 而在于注册方法

解决方法

在注册cell的时候, 如果是用的nib, 我用cell.classForCode作为reuseId就可以, 这里我猜测, 是只有在class第一次使用的时候, 会将class加载到内存中, 所以就可以了.
