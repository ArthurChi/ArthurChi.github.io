认识时间复杂度

衡量一个算法的好坏标准, 就是看一个算法的时间复杂度. 那么, 时间复杂度到底是什么呢? 怎么理解呢?

理解

如果拿着这个关键字去google, 会看到这样的描述:

算法的时间复杂度是一个函数，它定性描述该算法的运行时间。这是一个代表算法输入值的字符串的长度的函数。时间复杂度常用大O符号表述，不包括这个函数的低阶项和首项系数。

那么, 这里要注意, 不包括 低阶项 和 首项系数. 下面, 让我们通过一个具体的例子, 理解这个概念.

举例

为了理解这句话, 我们来写一个选择排序.

我们来分析这个过程:

  • i = 0, min记录i, j从i下一个开始遍历, 找到最小的, 并放到第一个 (假设arr长度为N, j经历了N次的比较)
  • i = 1, 那么这次j还是从i的下一个开始遍历, 那么这次j走了N-1
  • 一直到i走到最后, j的轨迹是一个等差数列, 公差为1
  • 等差数列的前N项和 Sn=n*a1+n(n-1)d/2
  • 整理后, 关于N的二次方程中, 最高项为N^2, 系数为d/2, 有一个一次向, 和一个常数项.
  • 根据上面的描述, 只要最高项, 且不要系数, 那么, 复杂度就位O(N^2)

拓展

另外, 我们发现, 上面的排序都是创建一个Array, 然后来操作数组进行的, 那么我们就可以直接使用对Array进行扩展, 来定义这些排序方法. 那么, 这样, 就解决了刚才的泛型的问题, 因为Array中, 有对泛型的使用, 从它的定义中, 可以看到使用了Element来接收泛型. 在这里, 我们对于泛型的操作是要比较他们的大小, 然后返回一个布尔型来描述结果, 那么, 我们就要求, 在我们的扩展中, 泛型要遵守Comparable协议.

递归的实质

对于通俗的说法, 递归就是自己调用自己. 但是, 这其中发生了很多事情. 比如我们要说的压栈.

一个例子

我们还是通过一个具体的例子来分析这个事情, 我们要得到一个数组的最大值

首先, 描述一下这个过程:

  1. 将数组分成两块, 左部分, 右部分.

  2. 然后得到左部分的最大值, 右部分的最大值
    a. 左边部分执行1.2步骤, 直到只剩下一个数, 直接返回

  3. 返回两个值的最大值

过程

当执行到let leftMax = maxValue(range: range.lowerBound..<rangeMid)这句的时候

  1. 当前函数执行的行数
  2. 当前函数的变量
  3. 当前函数信息

全都压入栈, 然后调用子过程. 进入子过程后, 发现, 仍然需要调用调用子过程, 那么就会再次进行上述的压栈, 直到剩下一个数了. 然后返回, 将刚才的栈进行弹栈. 如此往复, 直到执行完毕.

递归事件复杂度计算

通例:
T(N) = aT(n/b)+O(n^d)
n/b 为子过程的样本量
a为发生子过程的次数(这里并不是说总次数)
O(n^d) 为除去调用子过程之外的剩下的代价是多少

master公式

那么, 上述的这个公式就是master公式, 但是要注意, 应用这个公式的前提 一个过程中分的子过程的样本量是一样的, 在上述的例子中, 就是N/2. 另外, 通过这个判断复杂度的公式如下:

  1. log(b,a) > d 则复杂度为O(N^log(b,a))
  2. log(b,a) = d 则复杂度为O(N^d*logN)
  3. log(b,a) < d 则复杂度为O(N^d)

归并排序

复杂度分析

在归并中, 额外的操作复杂度(merge)N, 则d为1, 一次操作, 将整体分为了两个N/2, 则a和b均为2. 那么, 符合log(b,a) = d, d为1, 则复杂度为O(N*logN)

快速排序与一般优化手段

快速排序的思想

  • 一般是选中数组最后一个作为key
  • 然后从左边给一个小于区域的指针, 右边给一个大于区域的指针, 一个当前的遍历的指针
  • key的内容和current内容进行比较
    • 如果小于, 则小于区域的下一个和current交换, 小于区域扩大一, current向下移动一
    • 如果大于, 则大于区域的前一个和current交换, 大于区域扩大一
    • 当current和大于区域没有碰到, 就一直执行上面的过程
    • 返回current的位置
  • 对current的左边和右边递归执行上面的过程

tips: 注意递归过程中输入的range边界

优化一

当数组中有相等的情况, 就会一些浪费. 浪费点在于, 一次只能搞定一个数, 数组中的相等的部分会再次参与到下一次partion过程中去, 那么我们放出一个优化方案, 优化方式为返回current的位置的步骤, 改为返回大于区域和小于区域. 用来下次partion

tips: 注意递归过程中输入的range边界

优化二

对于上述的过程, 有一些反例, 例如, 数据情况为: [6, 5, 4, 3, 2, 1], 那么这个过程的复杂度就是O(N^2), 换句话说, 当前的过程和数据状况有关. 那么如何避免呢? 让我们引入随机快排

做法

随机快排的做法是在当前执行的范围内, 随机选择一个位置, 和最后一个数交换, 然后再执行经典快排的过程.

原理

如我们所看到的那样, 经典快排会受到数据状况的影响, 导致patition的过程分的很偏, 只有大于区域, 或这有小于区域, 那么, 如果我们让这个过程随机, 虽然也会出现上述情况, 但是从概率的长期期望来看, 复杂度就变成了O(NlogN)

稳定性

基于比较排序的算法稳定性

定义

如果在即将排序的的记录序列中, 有多个相同的关键字的记录, 若经过排序, 这些记录的相对次序保持不变, 就说这种排序是稳定的.

冒泡排序稳定性

网上的一些资料说冒泡是稳定的, 不过我觉得还是要看具体实现, 在我们的这个例子中, 取决于传入的比较规则, 如果规则是大于或者小于, 那么排序是稳定的, 如果规则是大于等于或者小于等于, 那么过程就是不稳定的了;

选择排序稳定性

根据数组实现的方式, 我们可以看到, 选择排序是不稳定的, 因为在找到最小值后, 要和无序部分的第一个进行交换.

插入排序稳定性

从无序部分拿出第一个数, 依次和有序部分比较, 直到大于等于或者小于等于就停止, 这样, 就可以实现为稳定的了

归并排序稳定性

归并在比较的时候, 可以做到稳定, 即相等元素不予交换.

快速排序稳定性

在partion的过程中, 在找大于区域的时候, 是一个不稳定的过程, 所以导致快排也是不稳定的; 理论上可以做到稳定(0 1 stable sort)

堆排序

在建立堆的过程, 就已经不是稳定了, 所以, 堆排序是不稳定的

稳定性的意义

假如我们有一个表格, 我们按照不同的列进行排序, 如果有的数据的有些项是相同的, 那么, 在第二次做排序的时候, 就可以保留上一次排序的结果.

id age score
1 23 60
2 25 60
3 23 70
4 23 50

例如, 我先按照年龄排序, 那么, id的顺序就是1, 3, 4, 2, 之后再按照得分排序, id就是4, 1, 2, 3; 1和2的相对位置没有变化

排序 稳定性 时间复杂度 空间复杂度
冒泡排序 Y O(N^2) O(1)
选择排序 N O(N^2) O(1)
插入排序 Y O(N^2) O(1)
归并排序 Y O(NxLogN) O(N)
快速排序 N(01 stable sort) O(NxLogN) O(LogN)
堆排序 N O(NxLogN) O(1)

工程排序

在工程中, 排序的选择, 应当遵循以下方式
当样本量小于60的时候, 选择插入排序
当样本量大于60的时候, 且样本为基础类型, 使用快速排序
当样本量大于60的时候, 且样本为复合类型, 使用归并排序

虽然插入排序的时间复杂度高, 但是常数项低, 在小样本情况下, 有优势
当样本为基础类型时, 无须考虑稳定性, 所以使用快速排序, 反之, 复合类型需要考虑稳定性, 所以使用归并, 没有使用堆排是因为堆排的常数项高于快排
